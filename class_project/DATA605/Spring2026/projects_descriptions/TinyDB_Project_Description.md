# TinyDB

## Description
- TinyDB is a lightweight, document-oriented database designed for Python
  applications, allowing for easy data storage and retrieval.
- It features a simple API that enables users to create, read, update, and
  delete (CRUD) operations on data stored as JSON documents.
- The database is schema-less, meaning you can store documents with varying
  structures without predefined schemas.
- TinyDB supports querying capabilities through a simple query language, making
  it easy to filter and manipulate data.
- It is designed for small to medium-sized projects, making it ideal for
  prototyping and educational purposes, without the overhead of larger
  databases.

## Project Objective
The goal of the project is to build a personal book collection management system
using TinyDB. Students will implement a system that allows users to add, update,
delete, and query their book collection, optimizing for efficient data retrieval
and user-friendly interaction.

## Dataset Suggestions
1. **Open Library Books**
   - **Source**: Open Library
   - **URL**: [Open Library API](https://openlibrary.org/developers/api)
   - **Data Contains**: Information about books including titles, authors,
     publication dates, and subjects.
   - **Access Requirements**: Free access, no authentication required.

2. **Goodreads Books**
   - **Source**: Goodreads (via public dataset on Kaggle)
   - **URL**:
     [Goodreads Books Dataset](https://www.kaggle.com/datasets/zygmunt/goodbooks-10k)
   - **Data Contains**: User ratings, book titles, authors, and genres.
   - **Access Requirements**: Free access, requires a Kaggle account to
     download.

3. **Project Gutenberg**
   - **Source**: Project Gutenberg
   - **URL**: [Project Gutenberg](https://www.gutenberg.org/)
   - **Data Contains**: Texts of public domain books, including metadata like
     title, author, and language.
   - **Access Requirements**: Free access, no authentication required.

4. **Books Dataset from Kaggle**
   - **Source**: Kaggle
   - **URL**: [Books Dataset](https://www.kaggle.com/datasets/shelleyj/books)
   - **Data Contains**: Information on various book titles, authors, genres, and
     publication years.
   - **Access Requirements**: Free access, requires a Kaggle account to
     download.

## Tasks
- **Set Up TinyDB**: Install TinyDB and set up the initial database structure
  for storing book information.
- **CRUD Operations**: Implement functions to create, read, update, and delete
  book entries in the TinyDB database.
- **Querying Data**: Develop functionality to query the database based on
  various filters (e.g., by author, genre, or publication year).
- **User Interface**: Create a simple command-line interface or GUI to allow
  users to interact with the book collection management system.
- **Data Analysis**: Analyze the book collection to provide insights, such as
  the most common authors or genres in the collection.

## Bonus Ideas
- **Recommendation System**: Extend the project by implementing a basic
  recommendation system that suggests books based on user preferences.
- **Data Visualization**: Integrate data visualization libraries to create
  graphs or charts that display trends in the book collection (e.g., publication
  year distribution).
- **Export/Import Functionality**: Add features to export the book collection to
  a CSV file or import data from a CSV file for easier management.
