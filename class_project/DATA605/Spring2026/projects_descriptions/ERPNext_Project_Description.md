# ERPNext

## Description
- ERPNext is an open-source enterprise resource planning (ERP) software that
  integrates various business processes into a single system.
- It features modules for accounting, inventory management, sales, purchasing,
  project management, and human resources, allowing for comprehensive data
  management.
- The tool supports customizable reporting and analytics, enabling users to
  generate insights from their business operations.
- ERPNext is built on the Frappe framework, which supports the creation of
  custom applications and extensions.
- It offers a user-friendly interface and is accessible via web browsers, making
  it easy to deploy and use without extensive technical knowledge.

## Project Objective
The goal of this project is to predict sales trends for a fictional retail
business using historical sales data. Students will optimize their models to
accurately forecast future sales for different products, thereby aiding
inventory management and marketing strategies.

## Dataset Suggestions
1. **Kaggle Store Sales Dataset**
   - **Source**: Kaggle
   - **URL**:
     [Store Sales - Time Series Forecasting](https://www.kaggle.com/c/store-sales-time-series-forecasting/data)
   - **Data Contains**: Daily sales data for a retail store, including product
     details, store information, and sales figures.
   - **Access Requirements**: Free account on Kaggle to download the dataset.

2. **UCI Machine Learning Repository - Online Retail Dataset**
   - **Source**: UCI Machine Learning Repository
   - **URL**:
     [Online Retail Dataset](https://archive.ics.uci.edu/ml/datasets/Online+Retail)
   - **Data Contains**: Transactions from a UK-based online retail store,
     including invoice numbers, product descriptions, quantities, and prices.
   - **Access Requirements**: Publicly accessible dataset, no authentication
     needed.

3. **Open Government Data - Monthly Retail Trade Report**
   - **Source**: U.S. Census Bureau
   - **URL**:
     [Monthly Retail Trade Report](https://www.census.gov/retail/index.html)
   - **Data Contains**: Monthly data on retail sales across various sectors in
     the U.S., including total sales, sales by category, and percentage changes.
   - **Access Requirements**: Publicly available data, no authentication
     required.

4. **Kaggle - Walmart Sales Forecasting**
   - **Source**: Kaggle
   - **URL**:
     [Walmart Sales Forecasting](https://www.kaggle.com/c/walmart-recruiting-store-sales-forecasting/data)
   - **Data Contains**: Historical sales data from Walmart stores, including
     store and department sales, promotions, and holiday information.
   - **Access Requirements**: Free account on Kaggle to download the dataset.

## Tasks
- **Data Exploration**: Analyze the chosen dataset to understand sales patterns,
  seasonal trends, and correlations between different variables.
- **Data Preprocessing**: Clean the dataset by handling missing values, removing
  duplicates, and transforming data types as needed for analysis.
- **Feature Engineering**: Create new features based on existing data, such as
  lagged sales, moving averages, and seasonal indicators to enhance model
  performance.
- **Model Selection and Training**: Choose appropriate machine learning models
  (e.g., linear regression, decision trees, or time series models) and train
  them on the preprocessed data.
- **Model Evaluation**: Assess the performance of the models using metrics like
  Mean Absolute Error (MAE) and Root Mean Squared Error (RMSE) to determine
  their accuracy in sales forecasting.
- **Visualization and Reporting**: Create visualizations to present findings,
  including sales trends and model predictions, and compile a report summarizing
  insights and recommendations.
