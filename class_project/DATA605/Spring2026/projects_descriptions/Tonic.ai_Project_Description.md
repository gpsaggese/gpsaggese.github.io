# Tonic.Ai

## Description
- Tonic.ai is a synthetic data generation tool that allows users to create
  realistic, privacy-preserving datasets for testing and development purposes.
- It helps data scientists and developers generate data that mimics real-world
  distributions while ensuring sensitive information is not exposed.
- The platform supports various data types, including structured data, time
  series, and text, enabling diverse applications across different domains.
- Tonic.ai integrates easily with popular data storage systems and can be used
  in conjunction with machine learning models to enhance data privacy during
  model training and testing.
- The tool provides an intuitive user interface and API for seamless data
  generation, allowing users to customize the data generation process based on
  specific requirements.

## Project Objective
The goal of this project is to create a synthetic dataset that mimics real-world
customer transaction data for a retail company. Students will build a machine
learning model to predict customer purchasing behavior based on various
features, optimizing for accuracy and interpretability.

## Dataset Suggestions
1. **Kaggle Retail Dataset**
   - **Source**: Kaggle
   - **URL**:
     [Retail Data](https://www.kaggle.com/datasets/carrie1/ecommerce-data)
   - **Data Contains**: Customer transactions, purchase amounts, product
     categories, and timestamps.
   - **Access Requirements**: Free account on Kaggle to download.

2. **UCI Machine Learning Repository - Online Retail**
   - **Source**: UCI
   - **URL**:
     [Online Retail Dataset](https://archive.ics.uci.edu/ml/datasets/online+retail)
   - **Data Contains**: Invoice data including product codes, quantities,
     prices, and customer IDs.
   - **Access Requirements**: Publicly accessible without authentication.

3. **Open Government Data - Retail Sales**
   - **Source**: Data.gov
   - **URL**: [Retail Sales Data](https://catalog.data.gov/dataset/retail-sales)
   - **Data Contains**: Monthly retail sales data across different sectors.
   - **Access Requirements**: No authentication required; freely available.

## Tasks
- **Data Exploration**: Analyze the provided datasets to understand the
  features, distributions, and relationships within the data.
- **Synthetic Data Generation**: Use Tonic.ai to create a synthetic dataset that
  reflects the characteristics of the original customer transaction data while
  ensuring privacy.
- **Machine Learning Model Development**: Build a classification model (e.g.,
  Decision Tree, Random Forest) to predict customer purchasing behavior based on
  the synthetic dataset.
- **Model Evaluation**: Assess the model's performance using metrics such as
  accuracy, precision, recall, and F1 score, and analyze feature importance.
- **Documentation and Presentation**: Prepare a report detailing the data
  generation process, model development, evaluation results, and insights gained
  from the project.

## Bonus Ideas
- **Compare Models**: Experiment with different machine learning algorithms
  (e.g., SVM, Neural Networks) to see how they perform on the synthetic dataset.
- **Data Augmentation**: Investigate techniques for augmenting the synthetic
  dataset to enhance model performance.
- **Real vs. Synthetic**: Conduct an analysis comparing model performance on
  real vs. synthetic datasets to understand the trade-offs of using synthetic
  data.
