# Amundsen

## Description
- **Data Discovery Tool**: Amundsen is an open-source data discovery and
  metadata engine designed to help users find and understand data within their
  organizations efficiently.
- **User-Friendly Interface**: It provides an intuitive web interface that
  allows users to search for datasets, view data lineage, and understand data
  usage across various projects.
- **Rich Metadata**: Amundsen captures and displays rich metadata, including
  data schema, data owners, and usage statistics, enabling users to make
  informed decisions on data utilization.
- **Integration Capabilities**: It integrates seamlessly with several data
  sources, including databases and data lakes, making it versatile for different
  data environments.
- **Community-Driven**: Being open-source, it has a growing community that
  contributes to its development, ensuring continuous improvement and new
  features.

## Project Objective
The goal of this project is to build a data catalog using Amundsen to facilitate
data discovery within a specific domain (e.g., healthcare, finance, or
e-commerce). Students will focus on optimizing the searchability and usability
of datasets by enriching the metadata and ensuring accurate data lineage is
displayed.

## Dataset Suggestions
1. **Kaggle Datasets**
   - **Source**: Kaggle
   - **URL**: [Kaggle Datasets](https://www.kaggle.com/datasets)
   - **Data**: Various open datasets on topics like health, finance, and social
     issues.
   - **Access Requirements**: Free account required to download datasets.

2. **UCI Machine Learning Repository**
   - **Source**: UCI Repository
   - **URL**: [UCI ML Repository](https://archive.ics.uci.edu/ml/index.php)
   - **Data**: A collection of datasets for machine learning, including health,
     finance, and social datasets.
   - **Access Requirements**: No authentication required; datasets can be
     downloaded freely.

3. **Open Government Data**
   - **Source**: Data.gov
   - **URL**: [Data.gov](https://www.data.gov/)
   - **Data**: Public datasets provided by the U.S. government on various topics
     such as health, education, and environment.
   - **Access Requirements**: No authentication required; datasets are publicly
     available.

4. **Hugging Face Datasets**
   - **Source**: Hugging Face
   - **URL**: [Hugging Face Datasets](https://huggingface.co/datasets)
   - **Data**: A variety of datasets including text, audio, and image data,
     suitable for different ML tasks.
   - **Access Requirements**: Free to use without authentication; datasets can
     be accessed through their API.

## Tasks
- **Setup Amundsen**: Install and configure Amundsen on your local machine or
  Google Colab environment, ensuring it connects to the chosen datasets.
- **Data Ingestion**: Ingest datasets into Amundsen by creating a metadata model
  that captures essential information such as schema, owners, and lineage.
- **Enrich Metadata**: Enhance the metadata by adding descriptions, tags, and
  usage statistics to improve data discoverability and usability.
- **Search Functionality**: Implement and test the search functionality to
  ensure users can easily find datasets based on keywords and filters.
- **User Testing**: Conduct user testing sessions to gather feedback on the
  usability of the Amundsen interface and make necessary adjustments based on
  this feedback.

## Bonus Ideas
- **Data Lineage Visualization**: Extend the project by adding a feature to
  visualize data lineage, showing how data flows through various
  transformations.
- **Integration with Other Tools**: Explore integrating Amundsen with other data
  tools (e.g., Apache Airflow or Jupyter Notebooks) to enhance the data
  workflow.
- **Comparative Analysis**: Compare the effectiveness of Amundsen with another
  data catalog tool (e.g., Apache Atlas) in terms of features and user
  experience.
- **Machine Learning Model Deployment**: Implement a simple machine learning
  model that predicts data quality based on metadata attributes.
